Single PC Setup

  1. get the input-overlay-ws server
  2. run it and right click the tray icon to open the settings
  3. copy your auth token
    4. (you can change it to whatever you like)
  4. paste the token in the auth token field above
  5. configure your overlay to your liking, then copy the URL and paste it as an OBS browser source
tip: you can configure the key whitelist in the server settings to ensure you are only sending keys over your network that are configured in the overlay

Sending keys to another pc

(eg. from gaming to streaming pc)
  1. edit config.json to set the address/port of your gaming pc
    run ipconfig in cmd and copy the local IPv4 address (usually 192.168.0.1 or 192.168.X.X with X being 0-255)
  2. enter the gaming pc's address in both the input-overlay-ws and configurator
  3. download the overlay as a local HTML file and add it as a browser source in OBS on your streaming pc instead of copying the url

key syntax

Comma-separated entries per row. Basic format:

[internal_name]:"[label]":[width class]

multi-key bindings

key_w|mouse_4:"W" - both W and mouse4 light up the same key
key_q|key_e:"use" - Q or E both trigger "use"

standard keys

label internal name
a-z key_a - key_z
0-9 key_0 - key_9
` - = [ ] \ ; ' , . / key_backtick, key_minus, key_equals…

special keys

label internal name
esc, tab, caps key_escape, key_tab, key_capslock
shift / ctrl / alt (L/R) key_leftshift, key_rightshift, etc.
space, enter, backspace key_space, key_enter, key_backspace
arrows key_leftarrow, key_uparrow, key_rightarrow, key_downarrow
f1-f12 key_f1 - key_f12
ins, del, home, end, pgup, pgdn key_insert, key_delete, key_home, key_end, key_pageup, key_pagedown
numpad 0-9 key_numpad_0 - key_numpad_9

mouse identifiers

label internal name
m1 (left) mouse_left
m2 (right) mouse_right
m3 (middle) mouse_middle
m4, m5 mouse_4, mouse_5
scroller scroller:"default":"up":"down"
side buttons mouse_side:"m5":"m4"
mouse pad mouse_pad:width:height:anchor mouse_pad:u4-25:u3-31:a-tl anchor classes:
top-left: a-tl,
top-right: a-tr,
bottom-left: a-bl,
bottom-right: a-br

width classes

u0-01 through u7 in 0.01u increments

special syntax

syntax effect
invisible / key_escape:"invis":u2 invisible spacer (supports width classes)
dummy empty row placeholder
key_a|key_b:"A" multi-key binding via pipe
